Fotoresistor (LDR)
Un extremo → 5V
Otro extremo → A1 (y resistencia 10kΩ a GND)
LEDs
LED Verde → Pin 8 (con resistencia 220Ω)
LED Amarillo → Pin 9 (con resistencia 220Ω)
LED Rojo → Pin 10 (con resistencia 220Ω)
Buzzer
Pin positivo → Pin 11
Pin negativo → GND

⚙
Lógica del Sistema

TEMPERATURA

  • Normal (18-25°C): LED verde encendido
  • Precaución (26-30°C): LED amarillo encendido
  • Crítico (>30°C): LED rojo + buzzer intermitente

LUMINOSIDAD

  • Si la luz es demasiado baja (<200), activar LED amarillo adicional

ALMACENAMIENTO

  • Guardar las últimas 5 lecturas de temperatura en un array
  • Cada 10 lecturas, calcular y mostrar el promedio en el Monitor Serie

Manejo de Arrays en Arduino

Los arrays (o arreglos) son estructuras que te permiten almacenar múltiples valores del mismo tipo en una sola variable. En este proyecto usás un array para guardar las últimas 5 temperaturas.

Declaración de Arrays

● Arduino C++ 
// Declarar array con tamaño fijo
float temperaturas[5];  // Array de 5 elementos float

// Declarar e inicializar con valores
int sensores[3] = {10, 20, 30};

// Declarar sin especificar tamaño (lo infiere)
float lecturas[] = {22.5, 23.1, 24.2};  // Tamaño = 3

Acceso a Elementos

● Arduino C++ 
float temps[5] = {22.5, 23.1, 24.2, 23.8, 22.9};

// Leer elemento (los índices empiezan en 0)
float primera = temps[0];     // 22.5
float ultima = temps[4];      // 22.9

// Modificar elemento
temps[2] = 25.0;  // Cambia el tercer elemento

// ¡CUIDADO! No existe temps[5] (solo 0-4)

Recorrer Arrays con FOR

● Arduino C++ 
// Ejemplo: Mostrar todas las temperaturas
float temps[5] = {22.5, 23.1, 24.2, 23.8, 22.9};

for (int i = 0; i < 5; i++) {
  Serial.print("Temp ");
  Serial.print(i + 1);
  Serial.print(": ");
  Serial.println(temps[i]);
}

// Ejemplo: Calcular promedio
float suma = 0;
for (int i = 0; i < 5; i++) {
  suma += temps[i];
}
float promedio = suma / 5.0;

// Ejemplo: Buscar máximo
float maximo = temps[0];
for (int i = 1; i < 5; i++) {
  if (temps[i] > maximo) {
    maximo = temps[i];
  }

Array Circular (Para este Proyecto)

La Cinta Transportadora Circular

Imaginá una cinta transportadora circular con 5 espacios. Cuando llega un nuevo item y todos los espacios están llenos, el item más viejo se cae y el nuevo toma su lugar. Así funciona tu array de temperaturas.

● Arduino C++ 
// Variables globales
float ultimas_temps[5];
int indice = 0;

void loop() {
  // Leer nueva temperatura
  float temp_actual = leerTemperatura();
  
  // Guardar en el array
  ultimas_temps[indice] = temp_actual;
  
  // Avanzar índice (vuelve a 0 cuando llega a 5)
  indice = (indice + 1) % 5;  // Operador módulo
  
  // Ahora ultimas_temps siempre tiene las últimas 5 lecturas
}
💡
El operador módulo (%)
indice % 5 devuelve el resto de dividir indice por 5.
Si indice es 0→0, 1→1, 2→2, 3→3, 4→4, 5→0, 6→1, 7→2...
¡Perfecto para hacer que el índice "dé la vuelta"!

Introducción a Programación Funcional

La programación funcional es un paradigma que trata los programas como evaluaciones de funciones matemáticas. Es el próximo tema que vamos a ver en profundidad.

¿Qué es un Paradigma?

Las Diferentes Formas de Resolver un Problema

Imaginá que necesitás calcular el promedio de 5 temperaturas:

  • Imperativo: "Tomá la primera, sumala, tomá la segunda, sumala..." (paso a paso)
  • Funcional: "Aplicá la función SUMAR a todos, después aplicá DIVIDIR" (composición de funciones)
  • Orientado a Objetos: "El objeto Termómetro tiene un método calcularPromedio()" (objetos y mensajes)

Conceptos Fundamentales

🔒
Funciones Puras
Una función que siempre devuelve el mismo resultado para los mismos parámetros y no produce efectos secundarios.
🧊
Inmutabilidad
Los datos no se modifican. En lugar de cambiar una variable, se crea una nueva con el valor modificado.
🎲
First-Class Functions
Las funciones son "ciudadanos de primera clase": se pueden asignar a variables, pasar como parámetros y retornar.
🔗
Composición
Combinar funciones simples para crear funciones más complejas. Como armar un sistema con piezas LEGO.

Ejemplo: Imperativo vs Funcional

Calculemos el promedio de temperaturas que superan 25°C:

● Python - Estilo IMPERATIVO 
# Estilo IMPERATIVO (paso a paso, modificando variables)
temperaturas = [22.5, 26.1, 24.2, 27.3, 23.8, 28.5]

suma = 0
contador = 0

for temp in temperaturas:
    if temp > 25.0:
        suma += temp        # Modificamos suma
        contador += 1      # Modificamos contador

if contador > 0:
    promedio = suma / contador
    print(f"Promedio: {promedio}")
else:
    print("No hay temperaturas altas")
● Python - Estilo FUNCIONAL 
# Estilo FUNCIONAL (composición de funciones, sin modificar variables)
temperaturas = [22.5, 26.1, 24.2, 27.3, 23.8, 28.5]

# Función pura: siempre devuelve lo mismo para el mismo input
def es_alta(temp):
    return temp > 25.0

# Composición de funciones
temps_altas = filter(es_alta, temperaturas)  # Filtrar
lista_altas = list(temps_altas)              # Convertir a lista

if lista_altas:
    promedio = sum(lista_altas) / len(lista_altas)
    print(f"Promedio: {promedio}")
else:
    print("No hay temperaturas altas")

# Versión más compacta con lambda (función anónima)
temps_altas = filter(lambda t: t > 25.0, temperaturas)

Funciones de Orden Superior

Son funciones que reciben otras funciones como parámetro o retornan funciones. Son fundamentales en programación funcional.

● Python - Funciones de Orden Superior 
# MAP: Aplica una función a cada elemento
sensores = [1023, 512, 768, 256]  # Valores analógicos 0-1023

# Convertir a porcentaje
def a_porcentaje(valor):
    return (valor / 1023.0) * 100

porcentajes = list(map(a_porcentaje, sensores))
print(porcentajes)  # [100.0, 50.05, 75.07, 25.02]

# FILTER: Filtra elementos que cumplen condición
def es_critico(porcentaje):
    return porcentaje > 80

criticos = list(filter(es_critico, porcentajes))
print(criticos)  # [100.0]

# REDUCE: Combina todos los elementos en uno solo
from functools import reduce

def sumar(a, b):
    return a + b

total = reduce(sumar, porcentajes)
print(f"Total: {total}")  # Total: 250.14

Ventajas de la Programación Funcional

Ventaja Explicación Código más legible Las funciones puras son más fáciles de entender (input → output) Más fácil de testear Las funciones puras siempre dan el mismo resultado Menos bugs Sin efectos secundarios, menos errores inesperados Reutilizable Funciones pequeñas y componibles son fáciles de reutilizar Paralelizable Sin estado compartido, más fácil ejecutar en paralelo
